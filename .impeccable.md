## Design Context

### Users
- Crypto wallet users (primarily mobile, Capacitor-based app)
- Privacy-conscious individuals using Nostr messaging
- Context: managing funds AND messaging in the same app
- Job to be done: send/receive private messages securely while managing wallet

### Brand Personality
- Secure, modern, trustworthy
- Professional but accessible
- 3 words: clean, confident, integrated
- Emotional goals: users should feel their communications are safe and the interface is reliable

### Aesthetic Direction
- Visual tone: Refined utility with subtle warmth
- Mobile-first messaging interface (WhatsApp/Telegram-like clarity)
- Light mode: soft blue-grey backgrounds (#f5f7fa), white cards, subtle shadows
- Dark mode: deep slate backgrounds (#0f172a), elevated surfaces (#1e293b)
- Primary accent: Blue (#3b82f6 → #2563eb gradient) used consistently for identity, sent bubbles, CTAs
- Avoid: pure black/white, neon glows, glassmorphism overload, generic AI gradients
- Theme-aware: must work with existing Quasar theme system and `getDarkModeClass()` pattern

### Design Principles
1. **Clarity over decoration** — every element should serve a functional purpose; no ornamental gradients or shadows
2. **Consistent depth** — use subtle shadows and borders to create hierarchy, not heavy cards inside cards
3. **Responsive to mode** — light and dark modes are first-class; never assume one mode
4. **Touch-first** — generous tap targets, clear active states, appropriate spacing for thumbs
5. **Crypto-native trust signals** — monospace for keys/IDs, clear status indicators, copy affordances
